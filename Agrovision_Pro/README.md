# Agrovision Pro: Hybrid AI Crop Diagnostician 🧬
**An advanced Two-Stage Artificial Intelligence platform capable of localizing and classifying 15 unique plant diseases with 99.2% accuracy.**

---

## 📖 Project Overview
Agrovision Pro is a full-stack system designed to bring commercial-grade agricultural AI to accessible consumer hardware. Unlike traditional ML-projects that rely on a single, often confused detection model, this project employs a **Two-Stage Hybrid Architecture**. By forcefully separating *spatial localization* (finding the sick spot) from *medical classification* (diagnosing the spot), the system bypasses the harsh limitations of object detection and achieves near-flawless analysis.

---

## 🛠️ The Technology Stack (What & Why)

### 1. Artificial Intelligence Core (The Brains)
*   **Ultralytics YOLOv8 (Stage 1 - The Locator)**
    *   *What:* A lightning-fast, single-shot spatial object detector.
    *   *Why:* When you feed an AI an entire leaf, it gets confused by the background (soil, sky, hands). We use YOLO strictly to scan the image for anomalous spots or burns and draw a box around them. We mathematically suppressed its classification abilities because it dropped to `0.85 mAP50` when trying to guess the diseases.
*   **PyTorch & EfficientNet-B0 (Stage 2 - The Specialist)**
    *   *What:* A highly-advanced convolutional neural network originally designed by Google, augmented with a custom 15-class linear output head.
    *   *Why:* YOLO struggled to tell the difference between "Tomato Early Blight" and "Tomato Late Blight". EfficientNet thrives on pure texture. By feeding EfficientNet the tiny cropped patches generated by YOLO, it pushed the system's diagnostic accuracy to a staggering **99.22%**.
*   **Scikit-Learn & Seaborn (Analytics)**
    *   *What:* Massive data-science metric calculators.
    *   *Why:* Accuracy alone is deceptive. We used these to calculate F1-Scores, Precision/Recall, and map out vivid graphical Heatmaps (Confusion Matrices) to mathematically prove the AI wasn't hallucinating.

### 2. Backend Engineering (The Router)
*   **FastAPI (Python)**
    *   *What:* An asynchronous server router handling the pipeline.
    *   *Why:* Standard frameworks (like Flask) lock up entirely while the GPU is thinking. FastAPI uses `async/await` byte streams so it can ingest high-res images instantly without stalling.
*   **OpenCV (cv2) & NumPy**
    *   *What:* C++ compiled image array manipulators.
    *   *Why:* Saving temporary files to the hard-drive is drastically too slow. OpenCV immediately translates the incoming data stream into raw `BGR` pixel matrices in RAM. It then uses YOLO's `[x1, y1, x2, y2]` coordinates to physically slice the matrix arrays out in microseconds.

### 3. Frontend Architecture (The Client)
*   **React.js & TypeScript**
    *   *What:* The highly-dynamic user interface.
    *   *Why:* The AI doesn't return one result; it returns an array of multiple localized spots. React natively maps these arrays into complex UI states cleanly. **TypeScript** was strictly used to prevent server-client type mismatches (ensuring floating-point confidence scores didn't crash JavaScript strings).
*   **Vite**
    *   *What:* The frontend compilation tool.
    *   *Why:* Vite replaces Webpack via native ES Modules, drastically improving hot-reload timing during complex algorithm iteration.
*   **HTML5 MediaDevices API (`navigator.mediaDevices`)**
    *   *What:* Direct web-browser hardware integrations.
    *   *Why:* We wanted to entirely eliminate clunky "File Explorer" uploads. This bypasses the OS, hooks directly into your laptop/phone's hardware camera, streams it onto a 2D HTML `<canvas>`, and snaps clinical frames straight into the API pipeline.

## 🔬 Dataset & Engineering Logistics
To train our specialist, we couldn't just use standard internet photos.
1. We utilized a primary dataset of **17,600 fully labeled photographs**.
2. We wrote a custom mathematical slicing script (`generate_crops.py`) that forced the computer to read the YOLO annotation arrays and cut out only the exact points of disease.
3. This algorithm synthesized a brand new, highly-specialized extraction dataset containing **45,228 localized micro-crop textures**, covering exactly 15 classes spanning 4 crop families (Tomato, Potato, Rice, Chilli, and Banana).

## ⚡ How to Run the Ecosystem

Because this relies on two heavy neural networks, the backend and frontend run decoupled.

### 1. Booting the Hybrid Pipeline (Backend)
Open a new terminal and navigate to your `detection_code2` directory.
```bash
pip install ultralytics torch torchvision opencv-python fastapi uvicorn python-multipart
python api.py
```
*(The server will boot on `localhost:8000` and load the `.pt` models into VRAM)*

### 2. Starting the Client (Frontend)
Open a second terminal and navigate to your `Agrovision_Pro` directory.
```bash
npm install
npm run dev
```
*(The UI will launch on `localhost:5173`. You can now trigger your hardware camera and run live disease diagnostics!)*
